# Codex + Claude Code 一键配置

先把压缩包“全部解压缩”到普通文件夹后再运行，不要直接在压缩包预览窗口里双击脚本。

这套安装文件会同时配置：

- `~/.codex/config.toml`
- `~/.codex/auth.json`
- `~/.claude/settings.json`

同时还会为 Claude Code 准备终端环境变量：

- Windows：生成 `.cmd` / `.ps1` 辅助文件
- macOS / Linux：生成 `~/.claude/claude-code-env.sh`，并自动写入常见 shell 配置文件

覆盖前会自动备份旧文件，格式如下：

- `原文件名.bak-YYYYMMDD-HHMMSS`

## 用户怎么操作

用户只需要做两步：

1. 按系统运行对应安装文件
2. 输入自己的两个令牌：
   `OPENAI_API_KEY`
   `ANTHROPIC_AUTH_TOKEN`

安装包不再内置任何用户令牌。

## Windows 用户

普通用户直接双击：

- `Windows-双击我-安装Codex和Claude配置.cmd`

如果要在终端中运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\Windows-install-codex-claude.ps1
```

安装完成后，建议关闭并重新打开：

- VS Code
- Claude Code
- Cursor
- Windsurf
- 终端窗口

## macOS 用户

终端运行：

```sh
sh ./macOS-Linux-终端版-安装Codex和Claude配置.sh
```

如果希望双击运行，先执行：

```sh
chmod +x ./macOS-双击我-安装Codex和Claude配置.command
```

然后双击：

- `macOS-双击我-安装Codex和Claude配置.command`

如果系统拦截，右键文件后选择“打开”，再允许一次即可。

## Linux 用户

终端运行：

```sh
sh ./macOS-Linux-终端版-安装Codex和Claude配置.sh
```

## 安装后会生成什么

### Codex

- `~/.codex/config.toml`
- `~/.codex/auth.json`

### Claude Code

- `~/.claude/settings.json`
- Windows：`~/.claude/claude-code-env.cmd`
- Windows：`~/.claude/claude-code-env.ps1`
- macOS / Linux：`~/.claude/claude-code-env.sh`

## 说明

- Windows 脚本会生成 Claude Code 的 CMD / PowerShell 环境辅助文件，Claude Code 本身使用 `~/.claude/settings.json`
- macOS / Linux 脚本会把 Claude Code 环境加载逻辑写入常见 shell 配置文件，重新打开终端后自动生效
- 如果之前已经有旧配置，脚本会先备份再覆盖
